The Trellis env story splits into three groups — _core Convex_, _auth-only_, and _identity-forwarding-only_. Don't set variables for features you're not using.

Fastest way to check a local setup is correct:

```bash [terminal]
pnpm dlx @lupinum/trellis doctor
```

The doctor statically checks the local config shape and reports missing variables for each feature without complaining about the ones that don't apply.

## Core Convex variables

Trellis needs a Convex deployment URL from either module config or an env var:

| Variable                 | Purpose                |
| ------------------------ | ---------------------- |
| `CONVEX_URL`             | Server-side Convex URL |
| `NUXT_PUBLIC_CONVEX_URL` | Client-side Convex URL |

Normal baseline:

```ts [nuxt.config.ts]
trellis: {
  url: process.env.CONVEX_URL,
}
```

If none of `trellis.url`, `CONVEX_URL`, or `NUXT_PUBLIC_CONVEX_URL` are set, Trellis can't talk to Convex.

## Convex site URL (HTTP Actions host)

| Variable                      | Purpose                         |
| ----------------------------- | ------------------------------- |
| `CONVEX_SITE_URL`             | HTTP Actions host — server-side |
| `NUXT_PUBLIC_CONVEX_SITE_URL` | HTTP Actions host — client-side |

::note
You usually don't set these. Trellis derives them automatically when `CONVEX_URL` points at a standard `.convex.cloud` deployment, or when you're using the normal local Convex ports (`127.0.0.1:3210` / `3211`). Set them explicitly only if you use a custom HTTP Actions domain or derivation fails.
::

## Auth variables

Only matter when `trellis.auth` is enabled:

| Variable             | Notes                                                                  |
| -------------------- | ---------------------------------------------------------------------- |
| `SITE_URL`           | App origin Better Auth uses for callbacks and trusted-origin checks    |
| `BETTER_AUTH_SECRET` | **Must match between local env and the Convex Better Auth deployment** |

::warning
Mismatched `BETTER_AUTH_SECRET` between local env and Convex is one of the most common auth bugs. Sign-in appears to work but sessions fail to validate. `trellis doctor` can catch missing local setup, but it cannot compare your local value to the secret configured in the Convex deployment.
::

## Trusted caller variables

Only matter when identity forwarding, webhook verification, or authenticated MCP flows are in use:

| Variable                         | Notes                                                 |
| -------------------------------- | ----------------------------------------------------- |
| `CONVEX_IDENTITY_FORWARDING_KEY` | Must be set both locally and in the Convex deployment |

Destructive MCP tools also choose a confirmation boundary. `backend` mode forwards the opaque confirmation token to Convex operation execution. `transport` mode verifies and redeems in the MCP server and therefore needs a distributed MCP `confirmationStore` in production.

Don't set these unless the app actually uses the corresponding surfaces.

## Local development defaults

Typical values:

```bash [.env.local]
CONVEX_URL=http://127.0.0.1:3210
CONVEX_SITE_URL=http://127.0.0.1:3211
SITE_URL=http://localhost:3000
```

In this repo's examples, the launcher injects `CONVEX_URL` and `CONVEX_SITE_URL` for you during `pnpm dev`.

## Launcher-owned vs. app-owned

| Launcher-owned (don't hand-edit) | App-owned                                 |
| -------------------------------- | ----------------------------------------- |
| `CONVEX_URL`                     | `SITE_URL`                                |
| `CONVEX_SITE_URL`                | `BETTER_AUTH_SECRET`                      |
| —                                | Provider creds (`GITHUB_CLIENT_ID`, etc.) |
| —                                | `CONVEX_IDENTITY_FORWARDING_KEY`          |

Manually overwriting launcher-owned values produces confusing local mismatches. If the launcher sets them, leave them alone.

## Common pitfalls

- Setting auth env vars when `auth: false` is the intended baseline.
- `BETTER_AUTH_SECRET` mismatch between local env and Convex deployment.
- Setting `CONVEX_SITE_URL` unnecessarily when Trellis could derive it.
- Editing launcher-owned local Convex URLs in example apps that already inject them.
