Reference for `trellis.permissions`. One key points `useAccess()` and `useAuthGuard()` at your backend-owned permission-context query. Permission codegen is off by default; the optional `codegen` flag adds tighter types and doctor metadata without changing runtime behavior.

## Two shapes

```ts [nuxt.config.ts]
trellis: {
  permissions: 'permissions/context.getAccessContext',
}
```

```ts [nuxt.config.ts]
trellis: {
  permissions: {
    query: 'permissions/context.getAccessContext',
    codegen: true,
  },
}
```

Both work. Use the shorthand for the common case; use the object when you need generated permission metadata or future options.

## What it wires

| Composable                     | Reads from                    |
| ------------------------------ | ----------------------------- |
| `useAccess()`                  | The configured query's result |
| `useAuthGuard({ permission })` | The `can` map in that result  |

## `codegen`

Optional, additive, and disabled unless explicitly enabled:

```ts [nuxt.config.ts]
trellis: {
  permissions: {
    query: 'permissions/context.getAccessContext',
    codegen: true,
  },
}
```

You can also pass an object when you need a non-canonical include list:

```ts [nuxt.config.ts]
trellis: {
  permissions: {
    query: 'permissions/context.getAccessContext',
    codegen: {
      include: ['convex/auth/permissions.ts'],
    },
  },
}
```

When enabled, Trellis emits:

- `.nuxt/types/trellis-permissions.d.ts` for typed permission-key/module augmentation
- `.nuxt/types/trellis-public-surface.d.ts` for generated operation, projection, tool, and recordAccess registries
- `.nuxt/trellis/permissions.json` for `trellis doctor` and devtools inspection
- `.nuxt/trellis/public-surface.json` for inspectable operation/tool inventory metadata

This does not generate runtime behavior and it does not try to analyze record-bound checks.

## Important constraint

::warning
The string must point at a real Convex query export path. If it doesn't, Trellis raises a _configured-function error_ at setup — it won't silently do nothing. Fix the path; the error tells you exactly which export it couldn't find.
::
