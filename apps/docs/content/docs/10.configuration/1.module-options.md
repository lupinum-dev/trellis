The root `trellis` config goes in `nuxt.config.ts`. Most apps fit the whole module setup in 10–15 lines — this page covers the top-level keys and points at the detail pages for the ones that have more surface.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  trellis: {
    url: process.env.CONVEX_URL,
    auth: { enabled: true },
    permissions: { query: 'permissions/context.getAccessContext' },
    mcp: { name: 'my-app', sessions: true },
  },
})
```

That's the common shape. The rest is opt-in.

## All top-level keys

| Key             | What it configures                          | Detail page                                                        |
| --------------- | ------------------------------------------- | ------------------------------------------------------------------ |
| `url`           | Convex deployment URL                       | [Environment variables](/docs/configuration/environment-variables) |
| `auth`          | Auth enablement + behavior                  | [Auth options](/docs/configuration/auth-options)                   |
| `permissions`   | Frontend permission-context query           | [Permissions options](/docs/configuration/permissions-options)     |
| `mcp`           | MCP runtime options                         | [MCP options](/docs/configuration/mcp-options)                     |
| `query`         | Default query behavior (SSR, subscriptions) | — (see below)                                                      |
| `upload`        | Upload defaults (concurrency)               | — (see below)                                                      |
| `observability` | Semantic observability config               | [Observability overview](/docs/observability/overview)             |
| `validation`    | Startup/build validation strictness         | — (see below)                                                      |

## `auth`

Two supported forms:

```ts
trellis: {
  auth: false
}
```

```ts
trellis: {
  auth: {
    enabled: true,
    routeProtection: { redirectTo: '/auth/signin' },
  },
}
```

See [Auth options](/docs/configuration/auth-options) for the full shape.

## `permissions`

Compact shorthand or object:

```ts
trellis: {
  permissions: 'permissions/context.getAccessContext'
}
```

```ts
trellis: {
  permissions: {
    query: 'permissions/context.getAccessContext',
  },
}
```

See [Permissions options](/docs/configuration/permissions-options).

## `query` defaults

```ts
trellis: {
  query: {
    server: true,
    subscribe: true,
  },
}
```

| Default           | Behavior                                      |
| ----------------- | --------------------------------------------- |
| `server: true`    | Queries run during SSR by default             |
| `subscribe: true` | Queries attach a live subscription by default |

Per-query options still override these. See [Queries](/docs/data-fetching/queries).

## `upload` defaults

```ts
trellis: {
  upload: { maxConcurrent: 3 },
}
```

Sets the default queue concurrency for `useConvexUpload()`. Per-call options override.

## `validation`

```ts
trellis: {
  validation: { strict: true },
}
```

When `strict` is on, module-validation warnings (missing tenant indexes, misconfigured paths) become startup or build errors. Off by default — turn it on for CI.
