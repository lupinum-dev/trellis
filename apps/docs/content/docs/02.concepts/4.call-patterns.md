---
title: 'Call patterns'
description: 'The normal Trellis call paths in one place.'
---

Trellis has a small set of app call paths. Use the direct path that matches the caller. Do not add bridge files, forwarded identity, MCP tools, or server wrappers when a protected Convex handler is enough.

## Browser reads

Public reads use public handlers:

```ts [convex/features/docs/domain.ts]
export const listPublicDocs = query.public({
  args: {},
  handler: async (ctx) => {
    return await ctx.db
      .query('docs')
      .withIndex('by_visibility', (q) => q.eq('visibility', 'public'))
      .collect()
  },
})
```

Signed-in reads use protected handlers and `appIdentity`:

```ts [convex/features/tasks/domain.ts]
export const listTasks = query.protected({
  args: {},
  guard: taskRead,
  handler: async (ctx) => {
    const appIdentity = await ctx.appIdentity()
    return await ctx.db
      .query('tasks')
      .withIndex('by_workspace', (q) => q.eq('workspaceId', appIdentity.workspaceId))
      .collect()
  },
})
```

## Browser writes

Use protected mutations for normal signed-in app work:

```ts [convex/features/tasks/domain.ts]
export const createTask = mutation.protected({
  args: createTaskArgs,
  guard: taskCreate,
  handler: async (ctx, args) => {
    const appIdentity = await ctx.appIdentity()
    return await ctx.db.insert('tasks', {
      title: args.title,
      ownerId: appIdentity.userId,
      workspaceId: appIdentity.workspaceId,
      createdAt: Date.now(),
    })
  },
})
```

`appIdentity.userId` is the local `users._id`. It is not a session profile id, provider subject, or `authKey`.

## Server calls

Nuxt server routes should use server Convex helpers:

```ts [server/api/tasks/export.get.ts]
export default defineEventHandler(async (event) => {
  return await serverConvexQuery(
    event,
    api.features.tasks.domain.listForExport,
    {},
    {
      auth: 'required',
    },
  )
})
```

Use trusted forwarding only after the route verified the external request:

```ts [server/api/webhook.post.ts]
await serverConvexMutation(event, api.features.tasks.webhooks.processTaskWebhook, payload, {
  auth: 'trusted',
  caller: {
    kind: 'service',
    serviceId: 'task-webhook',
    subject: subject.service('task-webhook'),
  },
  actingFor: await delegateToUser({
    userId: verifiedLocalUserId,
    allow: true,
    reason: 'verified task webhook',
  }),
})
```

Forwarded `user:*` subjects use local `users._id`.

## Permissions

Project permission state from Convex:

```ts [convex/permissions/context.ts]
export const getAccessContext = query.protected(
  defineAccessContext({
    resolve: getAppIdentity,
    permissions,
    extend: async (ctx, appIdentity) => {
      const user = await ctx.db.get(appIdentity.userId)
      return {
        email: user?.email ?? null,
        displayName: user?.displayName ?? null,
      }
    },
  }),
)
```

Then read it in Nuxt:

```ts [app/components/Toolbar.vue]
const { can } = useAccess()
const canCreateTask = can(taskCreate)
```

## Destructive operations

Use operations when one action needs the same preview, confirmation, execute path, tests, and MCP behavior:

```ts [convex/features/tasks/operations.ts]
export const deleteTaskOp = defineOperation({
  id: 'tasks.delete',
  kind: 'destructive',
  args: { id: v.id('tasks') },
  guard: taskDelete,
  preview: async (ctx, args) =>
    operationPreview({
      summary: 'Delete task',
      effects: [
        operationEffect({
          kind: 'delete',
          summary: 'Delete task',
          target: 'task',
          count: 1,
        }),
      ],
      confirm: { id: args.id },
    }),
  handler: async (ctx, args) => {
    await ctx.db.delete(args.id)
  },
})

export const previewDeleteTask = mutation.protected(previewOf(deleteTaskOp))
export const deleteTask = mutation.protected(deleteTaskOp)
```

## MCP projection

Reads can use `tool.query(...)`. Bounded writes can use `tool.mutation(...)`. Destructive or sensitive work should use `tool.operation(...)`:

```ts [server/mcp/tools/delete-task.ts]
export default tool.operation(deleteTaskOp, {
  execute: executeOperationRef(deleteTaskOp, api.features.tasks.operations.deleteTask),
  preview: previewOperationRef(deleteTaskOp, api.features.tasks.operations.previewDeleteTask),
  previewOperation: 'mutation',
})
```

Run this when the shape changes:

```bash
pnpm trellis doctor
pnpm trellis explain operation tasks.delete
```

Doctor should be able to show the exact MCP tool bound to the operation.
