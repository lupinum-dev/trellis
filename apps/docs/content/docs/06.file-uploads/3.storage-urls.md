You have a `storageId` saved on a document. You need a URL to stick in an `<img src>`. `useConvexStorageUrl` gives you exactly that — reactive, and auto-skipped when the id isn't ready yet.

```vue [app/components/Avatar.vue]
<script setup lang="ts">
import { api } from '#trellis/api'

const props = defineProps<{ storageId: string | null }>()
const url = useConvexStorageUrl(api.files.getUrl, () => props.storageId)
</script>

<template>
  <img v-if="url" :src="url" alt="avatar" />
</template>
```

## What this adds over `useConvexQuery`

It's a thin wrapper with one useful default: **if the id is `null` or `undefined`, the query is skipped automatically**. You don't need the usual skip guard around the args, and you don't get spurious fetches for rows that don't have a file yet.

## Reactive input, reactive output

Pass the id as a value, a `ref`, a `computed`, or a getter — whatever matches your source:

```ts [app/components/Avatar.vue]
// Plain value
useConvexStorageUrl(api.files.getUrl, user.avatarId)

// Getter (common with props)
useConvexStorageUrl(api.files.getUrl, () => props.storageId)

// Computed (when derived from other reactive state)
const selectedId = computed(() => activeDoc.value?.attachmentId ?? null)
useConvexStorageUrl(api.files.getUrl, selectedId)
```

When the id changes, the URL updates. When it becomes `null`, the URL goes back to `null` and no fetch runs.

## Common pairing with uploads

Upload and display on one reactive path:

```ts [app/components/AvatarEditor.vue]
const { upload, data: storageId } = useConvexUpload(api.files.generateUploadUrl)
const url = useConvexStorageUrl(api.files.getUrl, storageId)
```

As soon as the upload settles, `storageId` is set, the URL query runs, and the image shows up.

## Common pitfalls

- Wrapping `useConvexQuery` by hand and re-inventing the skip logic.
- Forgetting the URL can be `null` during load, skipped state, or when the file is missing. Always `v-if` the URL.
- Using the same pattern for non-file data — `useConvexStorageUrl` is specifically for storage IDs.
