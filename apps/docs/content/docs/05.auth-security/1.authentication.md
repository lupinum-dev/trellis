Turn on auth, hook Better Auth up through Convex, and read `isAuthenticated` / `sessionUser` in your components. Trellis owns the SSR handoff and the Convex auth refresh — the parts that are easy to get subtly wrong when you build them yourself.

::note
For the end-to-end flow ([appIdentity](/docs/concepts/glossary#appIdentity) resolution, protected query, ownership check), follow [Build a signed-in todo app](/docs/getting-started/build-a-signed-in-todo-app). This page is the auth runtime reference.
::

## Turn auth on

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@lupinum/trellis'],
  trellis: {
    url: process.env.CONVEX_URL,
    auth: { enabled: true },
  },
})
```

With auth enabled, Trellis expects the Better Auth bridge in `convex/`, a `users`
table that can mirror Convex identities, and three env vars set: `CONVEX_URL`,
`SITE_URL`, `BETTER_AUTH_SECRET`.

## Wire Better Auth through Convex

Four runtime files plus one schema invariant: install the Better Auth Convex
component, provide Better Auth config, create the Trellis bridge, register HTTP
routes, and make sure `users.authKey` is indexed by `by_auth_key`.

```ts [convex/convex.config.ts]
import betterAuth from '@convex-dev/better-auth/convex.config'
import { defineApp } from 'convex/server'

const app = defineApp()

app.use(betterAuth, { name: 'betterAuth' })

export default app
```

```ts [convex/auth.config.ts]
import { getAuthConfigProvider } from '@convex-dev/better-auth/auth-config'
import type { AuthConfig } from 'convex/server'

export default {
  providers: [getAuthConfigProvider()],
} satisfies AuthConfig
```

```ts [convex/auth.ts]
import { defineBetterAuth } from '@lupinum/trellis/auth'

import { components, internal } from './_generated/api'
import { mutation } from './_generated/server'
import authConfig from './auth.config'

const auth = defineBetterAuth(
  { components, internal, mutation, authConfig },
  { emailPassword: true },
)

export const authComponent = auth.authComponent
export const createAuth = auth.createAuth
// Internal bootstrap mutation used by the Trellis auth runtime.
export const createUserIfNeeded = auth.createUserIfNeeded
```

```ts [convex/http.ts]
import { httpRouter } from 'convex/server'

import { authComponent, createAuth } from './auth'

const http = httpRouter()
authComponent.registerRoutes(http, createAuth)

export default http
```

::tip
The official auth-enabled starters scaffold these files for you. Use `pnpm dlx @lupinum/trellis init <name> --template personal|workspace|workspace-mcp` when you want the paved path.
::

::note
`createUserIfNeeded` is an internal bootstrap mutation used by the Trellis auth runtime. App code should not call it manually.
::

```ts [convex/schema.ts]
users: defineTable({
  authKey: v.string(),
  email: v.optional(v.string()),
  displayName: v.optional(v.string()),
  createdAt: v.number(),
  updatedAt: v.number(),
}).index('by_auth_key', ['authKey'])
```

Trellis uses `identity.tokenIdentifier` as `users.authKey`. Your
`appIdentity` resolver can add more app fields, but it must be able to find the
mirrored user row by that auth key.

## When something breaks, run doctor first

`pnpm dlx @lupinum/trellis doctor` catches the three failure modes that account for most auth bugs: missing env vars, routes not registered, origin mismatches. Run it before debugging page code.

## Read auth state in components

`useConvexAuth` is the main composable. It's SSR-aware and hydrates without you wiring anything:

```ts [app/components/UserMenu.vue]
const { sessionUser, isAuthenticated, isPending, isAnonymous, signOut, refreshAuth } =
  useConvexAuth()
```

| Key               | What it is                                                                            |
| ----------------- | ------------------------------------------------------------------------------------- |
| `sessionUser`     | Session profile fields such as email and display name; it intentionally has no app id |
| `isAuthenticated` | `true` when signed in as a real account                                               |
| `isAnonymous`     | `true` for anonymous sessions (if enabled)                                            |
| `isPending`       | `true` while auth is initializing                                                     |
| `signOut()`       | Signs out + refreshes Convex auth                                                     |
| `refreshAuth()`   | Manually re-sync Convex auth state (rarely needed)                                    |

Use `useBetterAuthClient()` for direct Better Auth client calls. Do not pass `sessionUser` fields as app user ids; protected Convex handlers should use `appIdentity.userId`, which is the local `users._id`.

## Boundary components for templates

Three components that read better than `v-if` chains:

```vue [app/pages/index.vue]
<template>
  <ConvexAuthLoading>
    <p>Checking authentication…</p>
  </ConvexAuthLoading>

  <ConvexUnauthenticated>
    <LoginForm />
  </ConvexUnauthenticated>

  <ConvexAuthenticated>
    <Dashboard />
  </ConvexAuthenticated>
</template>
```

Use them when the template needs three branches — loading, signed out, signed in — without repeating `v-if="isAuthenticated && !isPending"` six times.

## Sign-in and sign-up actions

Don't call the Better Auth client directly and try to remember to refresh Convex auth afterwards. Use `useBetterAuthActions` — it wraps the call, refreshes Convex, tracks state, and handles redirects.

```ts [app/components/LoginForm.vue]
const client = useBetterAuthClient()
const authAction = useBetterAuthActions()

async function handleSignIn(email: string, password: string) {
  if (!client) throw new Error('Auth client unavailable.')

  await authAction.execute(() => client.signIn.email({ email, password }), { redirectTo: '/' })
}
```

The action gives you tracked state like mutations (`status`, `pending`, `error`, `data`, `reset`), so you can disable the submit button and surface errors without a separate ref. Unlike mutation calls, `execute()` does not throw on auth failure; it sets `error` and returns `undefined`.

::warning
If you skip `useBetterAuthActions` and call `client.signIn.email(...)` directly, the Better Auth session updates but Convex auth doesn't refresh. Any protected query on the next tick will still see the caller as signed out.
::

## Common pitfalls

- Calling Better Auth directly instead of going through `useBetterAuthActions`.
- Missing `SITE_URL` or `BETTER_AUTH_SECRET` — doctor catches both.
- Installing the Better Auth component package but not adding it to `convex/convex.config.ts`.
- Importing `./auth.config` from `convex/auth.ts` without creating `convex/auth.config.ts`.
- Registering auth routes in `convex/auth.ts` but forgetting `convex/http.ts`.
- Missing the `users.by_auth_key` index that `createUserIfNeeded` and `appIdentity` resolution rely on.
- Building an auth bootstrap plugin when `useConvexAuth` already owns the SSR handoff.
