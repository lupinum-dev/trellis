Build the smallest Trellis app that feels real: render a list on the page, add a record, and watch the list update with no refetch code anywhere.

This guide gives you one backend contract, one edge schema, one public query, one public mutation, one Nuxt page rendering the list, and one live update you can verify yourself. It follows the same shape as [`examples/01-public-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/01-public-todo).

::note
You need a working installation first. If you haven't done that yet, go to [Installation](/docs/getting-started/installation). For this smallest path, the only required env value is `CONVEX_URL`.
::

::tip
If you want the paved generated shape instead of the hand-built walkthrough, start with `pnpm dlx @lupinum/trellis init my-app --template public` and compare this page against the generated files. This guide uses the same feature-first layout as the maintained example and starter.
::

## Build it

::steps{level="3"}

### Define the edge schema

Put the runtime-neutral contract in `shared/features/todos/contract.ts`. That keeps the Convex args, derived Zod schema, and metadata in one place.

```ts [shared/features/todos/contract.ts]
import { defineArgs } from '@lupinum/trellis/args'
import { v } from 'convex/values'

export const createTodo = defineArgs({
  description: 'Create a public todo item',
  args: {
    title: v.string(),
  },
})
```

### Add the root schema and Trellis runtime

Convex still needs one root schema file and one runtime builder file even when feature folders own the details.

```ts [convex/schema.ts]
import { defineSchema } from 'convex/server'

import { todosTables } from './features/todos/schema'

export default defineSchema({
  ...todosTables,
})
```

```ts [convex/functions.ts]
import { defineTrellis } from '@lupinum/trellis/backend'

import { mutation as generatedMutation, query as generatedQuery } from './_generated/server'

export const { mutation, query } = defineTrellis({
  query: generatedQuery,
  mutation: generatedMutation,
})
```

### Add the smallest Trellis handlers

Wire `defineTrellis` so you get `mutation` and `query` builders, then define a list, a create, and a toggle. Every handler uses the explicit public lane.

```ts [convex/features/todos/domain.ts]
import { requireRecord } from '@lupinum/trellis/auth'
import { v } from 'convex/values'

import { createTodo } from '../../../shared/features/todos/contract'
import { mutation, query } from '../../functions'

export const list = query.public({
  args: {},
  handler: async (ctx) => {
    return await ctx.db.query('todos').order('desc').collect()
  },
})

export const create = mutation.public({
  args: createTodo.args,
  handler: async (ctx, args) => {
    return await ctx.db.insert('todos', {
      title: args.title,
      completed: false,
      createdAt: Date.now(),
    })
  },
})

export const toggle = mutation.public({
  args: { id: v.id('todos') },
  load: async (ctx, args) => {
    const todo = await ctx.db.get(args.id)
    requireRecord(todo, 'Todo')
    return { todo }
  },
  handler: async (ctx, args, { todo }) => {
    await ctx.db.patch(args.id, {
      completed: !todo.completed,
    })
  },
})
```

This is the whole point of the first live query path: Trellis stays small when the app is small.

### Add the feature table

The handlers above read and write `todos`, so the feature also needs its table definition:

```ts [convex/features/todos/schema.ts]
import { defineTable } from 'convex/server'
import { v } from 'convex/values'

export const todosTables = {
  todos: defineTable({
    title: v.string(),
    completed: v.boolean(),
    createdAt: v.number(),
  }),
}
```

### Render the query on a page

Use `useConvexQuery` for the list and `useConvexMutation` for creation.

```vue [app/pages/index.vue]
<script setup lang="ts">
import { computed, ref } from 'vue'

import { api } from '#trellis/api'
import { createTodo as createTodoContract } from '~/shared/features/todos/contract'

const { data: todos, pending, error } = await useConvexQuery(api.features.todos.domain.list, {})
const createTodoMutation = useConvexMutation(api.features.todos.domain.create)

const title = ref('')
const todoItems = computed(() => todos.value ?? [])

async function handleCreate() {
  const parsed = createTodoContract.zod.safeParse({ title: title.value })
  if (!parsed.success) return

  await createTodoMutation(parsed.data)
  title.value = ''
}
</script>

<template>
  <form @submit.prevent="handleCreate">
    <input v-model="title" placeholder="Write something small and concrete" required />
    <button type="submit">Add</button>
  </form>

  <p v-if="error">{{ error.message }}</p>
  <p v-else-if="pending">Loading...</p>
  <ul v-else>
    <li v-for="todo in todoItems" :key="todo._id">
      {{ todo.title }}
    </li>
  </ul>
</template>
```

The important thing is what you _don't_ have to write — no fetch endpoint, no manual refetch after create, no transport glue between Nuxt and Convex. `useConvexQuery` returns a live subscription, so when the mutation commits, the list updates on its own.

### Verify the live update

Start Convex in one terminal and Nuxt in another:

```bash [terminal]
pnpm exec convex dev
pnpm dev
```

Then add a todo. You should see:

- the page render the initial list on load (from SSR);
- the form create a todo when submitted;
- the list update without a manual refresh.

That's the first real Trellis win.

::

## Common pitfalls

- Adding sign-in or protected data rules before the public baseline is working.
- Treating `useConvexQuery` like a one-shot fetch and manually refetching after writes — it's already live.
- Forgetting the root `convex/schema.ts` or `convex/functions.ts` files that make the feature files reachable.
- Hiding the edge schema in the page instead of keeping it in `shared/`.
- Hiding the protected contract inside a big handler file instead of keeping it beside the backend module.
