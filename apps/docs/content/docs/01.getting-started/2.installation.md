Install the module, hand it a Convex deployment URL, and run one command to verify everything connected. That is the whole installation story. The rest of this page covers which baseline to pick so the app starts on the right Trellis shape instead of accreting setup by accident.

This page does not cover app-owned auth code, access context, or MCP tools. Those come after you have a healthy baseline.

## Prerequisites

You'll need:

- Nuxt 4
- Node 18.18 or newer
- pnpm 10.33 or newer through Corepack
- a Convex project (or a local Convex dev environment)

::note
Prefer a known-good starting point to a blank app? Clone any of the [runnable examples](https://github.com/lupinum-dev/trellis/tree/main/examples) and skip this page.
::

## Install the package

::code-group

```bash [pnpm]
pnpm add convex @lupinum/trellis
```

```bash [npm]
npm install convex @lupinum/trellis
```

```bash [yarn]
yarn add convex @lupinum/trellis
```

```bash [bun]
bun add convex @lupinum/trellis
```

::

Then register the module in `nuxt.config.ts` and choose the baseline that matches the app you are actually building.

## Pick a baseline

Trellis enables auth by default. That is right for most real Trellis apps, but wrong for a public-only baseline, so make the choice explicitly on the first run.

::tabs

:::tabs-item{label="Public-only" icon="i-lucide-globe"}

Use this when you want a public-first starting point with queries and mutations only.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@lupinum/trellis'],
  trellis: {
    url: process.env.CONVEX_URL,
  },
})
```

:::

:::tabs-item{label="Auth-ready" icon="i-lucide-lock"}

Use this when the app is meant to start on the protected path from day one.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@lupinum/trellis'],
  trellis: {
    url: process.env.CONVEX_URL,
    auth: {
      enabled: true,
    },
  },
})
```

:::

::

## Environment variables

### Required

`CONVEX_URL` — the deployment URL Trellis will talk to. Set it in your `.env` and reference it from the module config as above.

::note
You can also use `NUXT_PUBLIC_CONVEX_URL`, but `CONVEX_URL` read through `trellis.url: process.env.CONVEX_URL` is the clearest baseline.
::

### Required when auth is enabled

Two more values, both non-optional once auth is on:

- `SITE_URL` — for Better Auth callbacks and trusted-origin checks.
- `BETTER_AUTH_SECRET` — set in both your local environment _and_ the Convex Better Auth deployment. They must match.

### Advanced: custom HTTP Actions domain

Trellis derives the Convex HTTP Actions host from `CONVEX_URL` for standard `.convex.cloud` URLs and local `127.0.0.1:3210` dev setups. Skip this section unless that derivation doesn't work for you.

If you're on a custom HTTP Actions domain, set one of these explicitly:

- `CONVEX_SITE_URL`
- `NUXT_PUBLIC_CONVEX_SITE_URL`

## Verify the setup

Run the built-in doctor from the app root:

```bash [terminal]
pnpm dlx @lupinum/trellis doctor
```

The output tells you what to fix:

- **Public-only app** — confirm auth is unset or `auth: false` so auth warnings aren't flagged.
- **Auth-enabled app** — confirm `SITE_URL`, `BETTER_AUTH_SECRET`, and Better Auth route registration are in place.
- **Missing Convex URL** — always fix this before writing app code. Nothing will work until it's set.

::warning
Don't skip `trellis doctor` after an auth or env failure. Chasing symptoms before the doctor passes is how people lose an afternoon to problems Trellis would have pointed at directly.
::

## What counts as a successful installation

At the installation stage, success is narrower than “the app works.” You are proving that the Trellis platform wiring is healthy before you start adding app code.

You are done with this page when:

1. `@lupinum/trellis` is installed and registered in `nuxt.config.ts`;
2. `CONVEX_URL` is set;
3. `pnpm dlx @lupinum/trellis doctor` passes for the baseline you picked;
4. the Nuxt app starts without Trellis config errors.

The first real app success comes on the next page, where you will add the Convex schema, backend handlers, and page code together in one sequence.

If you used `pnpm dlx @lupinum/trellis init <name> --template ...`, read [Canonical app layout](/docs/getting-started/canonical-app-layout) before reorganizing anything. The generator now gives you the intended starter shape, and moving away from it early usually creates avoidable app-to-app drift.

## A peek at the shape

Here's what the Nuxt side of a minimal Trellis app looks like — one query, one mutation. You'll build these (plus the matching Convex handlers) on the next page.

::code-group

```vue [Query]
<script setup lang="ts">
import { api } from '#trellis/api'

const { data: tasks, pending, error } = await useConvexQuery(api.tasks.list, {})
</script>

<template>
  <p v-if="error">{{ error.message }}</p>
  <p v-else-if="pending">Loading…</p>
  <ul v-else>
    <li v-for="task in tasks" :key="task._id">{{ task.title }}</li>
  </ul>
</template>
```

```vue [Mutation]
<script setup lang="ts">
import { ref } from 'vue'
import { api } from '#trellis/api'

const createTask = useConvexMutation(api.tasks.create)
const title = ref('')

async function handleAdd() {
  await createTask({ title: title.value })
  title.value = ''
}
</script>

<template>
  <form @submit.prevent="handleAdd">
    <input v-model="title" required />
    <button type="submit" :disabled="createTask.pending.value">Add</button>
  </form>
  <p v-if="createTask.error.value">{{ createTask.error.value.message }}</p>
</template>
```

::

## Common pitfalls

- Auth left enabled while trying to build a public-only baseline (you'll see auth errors you don't expect).
- Setting `CONVEX_URL` manually inside an example that already injects local Convex URLs for you.
- Treating `trellis doctor` as optional after a failure.
- Starting from the workspace example when the app only needs public or personal scope.
