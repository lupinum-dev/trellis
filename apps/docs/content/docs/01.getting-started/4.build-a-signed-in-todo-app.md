Add sign-in, one protected query, one protected mutation. At the end you'll have a todo app where each user sees only their own list — and if you sign in as a second account, the first account's todos stay invisible.

The scope stays narrow on purpose. No tenancy, no access context, no MCP — just the smallest protected app that proves the boundary works. The code follows [`examples/02-auth-todo`](https://github.com/lupinum-dev/trellis/tree/main/examples/02-auth-todo).

::note
Do the [First live query](/docs/getting-started/first-live-query) before this page. Protection is easier to understand when you've already seen the unprotected baseline.
::

::tip
If you want the paved Trellis shape, run `pnpm dlx @lupinum/trellis init my-app --template personal` first. The generated starter puts feature code under `convex/features/`, runtime-neutral contracts under `shared/features/`, and app-owned appIdentity logic under `convex/auth/`. This page uses that same feature-first layout.
::

## Before you start

You should already have the installation baseline. For the auth-enabled path, these values must all be set:

- `CONVEX_URL`
- `SITE_URL`
- `BETTER_AUTH_SECRET`

If `pnpm dlx @lupinum/trellis doctor` isn't clean, stop and fix it first — debugging an auth app on top of a broken baseline is a bad time.

## Build it

::steps{level="3"}

### Enable the module auth path

Start from the auth-ready module config:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@lupinum/trellis'],
  trellis: {
    url: process.env.CONVEX_URL,
    auth: {
      enabled: true,
    },
  },
})
```

That puts Trellis on the signed-in path from the first run.

### Register Better Auth in Convex

Three files here: install the Better Auth component, create the auth bridge, and register the HTTP routes so Better Auth can serve its callbacks.

```ts [convex/convex.config.ts]
import betterAuth from '@convex-dev/better-auth/convex.config'
import { defineApp } from 'convex/server'

const app = defineApp()

app.use(betterAuth, { name: 'betterAuth' })

export default app
```

```ts [convex/auth.ts]
import { defineBetterAuth } from '@lupinum/trellis/auth'

import { components, internal } from './_generated/api'
import { mutation } from './_generated/server'
import authConfig from './auth.config'

const auth = defineBetterAuth(
  { components, internal, mutation, authConfig },
  { emailPassword: true },
)

export const authComponent = auth.authComponent
export const createAuth = auth.createAuth
// Internal bootstrap mutation used by the Trellis auth runtime.
export const createUserIfNeeded = auth.createUserIfNeeded
```

```ts [convex/http.ts]
import { httpRouter } from 'convex/server'

import { authComponent, createAuth } from './auth'

const http = httpRouter()

authComponent.registerRoutes(http, createAuth)

export default http
```

`createUserIfNeeded` is the internal bootstrap mutation used by the Trellis auth runtime. It creates or updates the app-owned `users` row from Convex's verified identity, using `identity.tokenIdentifier` as `users.authKey`. App code should not call it manually.

::tip
The official starters scaffold the default auth files if you'd rather not type them by hand.
::

### Resolve the current signed-in user

For this first protected app, Trellis needs one small piece of app-owned identity: "which signed-in user is this?" Later docs call that shape the `appIdentity`. On this page, keep it simpler: resolve the current app user from the mirrored `users` table and return the smallest shape the handlers need.

```ts [convex/auth/appIdentity.ts]
import { getAuth } from '@lupinum/trellis/auth'
import type { GenericActionCtx, GenericMutationCtx, GenericQueryCtx } from 'convex/server'

import type { DataModel, Id } from '../_generated/dataModel'

export type AppIdentity = { kind: 'user'; userId: Id<'users'>; authKey: string } | null

type AuthTodoCtx =
  | GenericQueryCtx<DataModel>
  | GenericMutationCtx<DataModel>
  | GenericActionCtx<DataModel>

export async function getAppIdentity(ctx: AuthTodoCtx): Promise<AppIdentity> {
  const auth = await getAuth(ctx)
  if (!auth) return null
  if (!('db' in ctx)) {
    throw new Error('AuthTodo appIdentity resolution requires a query or mutation context.')
  }

  const user = await ctx.db
    .query('users')
    .withIndex('by_auth_key', (q: any) => q.eq('authKey', auth.authKey))
    .first()

  if (!user) {
    throw new Error(
      [
        \`Expected a Trellis users row for auth key "\${auth.authKey}", but none was found.\`,
        'Verify convex/auth.ts exports createUserIfNeeded and the Trellis auth bootstrap is enabled.',
      ].join(' '),
    )
  }

  return {
    kind: 'user',
    userId: user._id,
    authKey: user.authKey,
  }
}
```

Then wire that user resolver into Trellis:

```ts [convex/functions.ts]
import { defineTrellis } from '@lupinum/trellis/backend'

import { mutation as generatedMutation, query as generatedQuery } from './_generated/server'
import { getAppIdentity } from './auth/appIdentity'

export const { mutation, query, raw } = defineTrellis(
  { query: generatedQuery, mutation: generatedMutation },
  {
    appIdentity: getAppIdentity,
  },
)
```

This is the first meaningful boundary: Trellis owns the protected function pipeline, but your app decides what a valid signed-in user looks like.

### Add one signed-in check, one table, two handlers

Define the smallest useful outer check: "is this caller signed in?" Then add a schema and two protected handlers.

```ts [convex/auth/guards.ts]
import { defineGuard } from '@lupinum/trellis/auth'

import type { AppIdentity } from './appIdentity'

export const isAuthenticated = defineGuard<AppIdentity>(
  'Authenticated',
  (appIdentity) => appIdentity !== null,
)
```

```ts [convex/schema.ts]
import { defineSchema, defineTable } from 'convex/server'
import { v } from 'convex/values'

export default defineSchema({
  users: defineTable({
    authKey: v.string(),
    email: v.optional(v.string()),
    displayName: v.optional(v.string()),
    createdAt: v.number(),
    updatedAt: v.number(),
  }).index('by_auth_key', ['authKey']),

  todos: defineTable({
    ownerId: v.id('users'),
    title: v.string(),
    completed: v.boolean(),
    createdAt: v.number(),
  }).index('by_owner', ['ownerId']),
})
```

```ts [convex/features/todos/domain.ts]
import { createTodo } from '../../../shared/features/todos/contract'
import { isAuthenticated } from '../../auth/guards'
import { mutation, query } from '../../functions'

export const list = query.protected({
  args: {},
  guard: isAuthenticated,
  handler: async (ctx) => {
    const appIdentity = await ctx.appIdentity()

    return await ctx.db
      .query('todos')
      .withIndex('by_owner', (q) => q.eq('ownerId', appIdentity.userId))
      .order('desc')
      .collect()
  },
})

export const create = mutation.protected({
  args: createTodo.args,
  guard: isAuthenticated,
  handler: async (ctx, args) => {
    const appIdentity = await ctx.appIdentity()

    return await ctx.db.insert('todos', {
      ownerId: appIdentity.userId,
      title: args.title,
      completed: false,
      createdAt: Date.now(),
    })
  },
})
```

One guard protects both handlers. Ownership is explicit in the stored row — which is what makes the isolation check below possible.

### Build the Nuxt page

Use `useConvexAuth` for session state, `useBetterAuthActions` for sign-in transitions, `useConvexQuery` for the protected list, and `useConvexMutation` for creation. Pass `undefined` as query args when the user is not signed in yet. That tells Trellis to skip the subscription entirely.

```vue [app/pages/index.vue]
<script setup lang="ts">
import * as z from 'zod'

import { api } from '#trellis/api'

const { isAuthenticated, isPending, sessionUser, signOut } = useConvexAuth()
const client = useBetterAuthClient()
const authAction = useBetterAuthActions()

const title = ref('')
const createTodo = useConvexMutation(api.features.todos.domain.create)

const todoArgs = computed(() => (isAuthenticated.value ? {} : undefined))

const {
  data: todos,
  pending: todosPending,
  error: todosError,
} = await useConvexQuery(api.features.todos.domain.list, todoArgs)

const signInSchema = z.object({
  email: z.string().email(),
  password: z.string().min(1),
})

async function handleSignIn(payload: { email: string; password: string }) {
  if (!client) throw new Error('Auth client unavailable.')
  await authAction.execute(() => client.signIn.email(payload), { redirectTo: '/' })
}

async function handleCreateTodo() {
  await createTodo({ title: title.value })
  title.value = ''
}
</script>
```

::note
The complete page in the example also covers sign-up, sign-out, loading states, toggle, and delete. Keep your first pass smaller if you don't need those yet — adding them later is mechanical.

## What success looks like

You're done when you can:

1. create an account;
2. sign in without writing auth glue code in the page;
3. see an empty personal list;
4. create a todo and see it appear;
5. sign in as a different account and confirm the first account's todo **does not appear**.

::caution
That last check is the whole point of this tutorial. If the second account can see the first account's todos, protection is happening in the UI only — which isn't protection. The rest of this tutorial assumes you ran it.
::

## Common pitfalls

- Auth enabled but `SITE_URL` or `BETTER_AUTH_SECRET` missing.
- Better Auth routes defined in `convex/auth.ts` but never registered in `convex/http.ts`.
- Forgetting `createUserIfNeeded` in `convex/auth.ts` — sign-in works, but the app-owned `users` row never lands and current-user resolution fails with a setup error.
- The page subscribes to the protected query before auth is ready (fix: gate the args on `isAuthenticated`).
- The handler writes rows but doesn't store `ownerId`, so the list query can't scope correctly.
