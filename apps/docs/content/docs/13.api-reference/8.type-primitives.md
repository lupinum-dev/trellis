title: 'Type Primitives'
description: 'Advanced TypeScript helpers for validating ids, options, and generated registries against Trellis’s public type surface.'
---

`@lupinum/trellis/type-primitives` is the supported advanced TypeScript entrypoint for Trellis.

Use it when you need to validate ids or option shapes against the generated Trellis type surface without importing repo-internal helpers.

## What lives here

- `ValidateOperationId` and `ValidateRegisteredOperationId`
- `ValidateOperationProjection`
- `ValidatePermissionKey`
- `ValidateAccessKey`
- `ValidateToolArgs`
- `ValidateMcpToolOptions`
- `InferOperationResult`
- `InferAccessContext`
- `OperationsById`, `OperationExecutionsById`, `OperationPreviewsById`, and `ToolsByName`

## Generated registries

When `trellis.permissions.codegen` is enabled, Trellis emits additive declarations under:

- `.nuxt/types/trellis-permissions.d.ts`
- `.nuxt/types/trellis-public-surface.d.ts`

Those declarations register:

- permission and projected-permission keys
- MCP recordAccess keys
- operation ids
- execute and preview projection maps
- MCP tool names

This is what makes helpers such as `ValidateRegisteredOperationId<'tasks.archive'>` or `ToolsByName['archive-task']` narrow in the consuming app.

## Example

```ts
import type {
  OperationExecutionsById,
  RegisteredToolName,
  ToolsByName,
  ValidateRegisteredOperationId,
  ValidateToolName,
} from '@lupinum/trellis/type-primitives'

type OperationId = ValidateRegisteredOperationId<'tasks.archive'>
type ToolName = ValidateToolName<'archive-task'>

type ExecuteRef = OperationExecutionsById[OperationId]
type ToolDef = ToolsByName[ToolName]
```

## What Not To Rely On

- repo-internal module paths
- internal utility carriers or scaffolding helpers
- generated file internals beyond the documented `.nuxt/types/*` outputs

Use the package entrypoints and generated public registries, not source-path imports.

## Public Type Verification

The public type surface is verified against:

- TypeScript `5.9.3`
- TypeScript `6.0.2`

That verification is part of Trellis repo checks through the public entrypoint `dts` suite, not through a compatibility lane for old APIs.
