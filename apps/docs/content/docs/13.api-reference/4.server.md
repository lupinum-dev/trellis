Reference for the Trellis helpers you use from Nitro routes, middleware, and other Nuxt server files. Each helper runs the same backend authorization path as a browser call — only the transport changes.

## `serverConvexQuery(event, fn, args?, options?)`

Run a protected Convex query from a Nitro server handler.

**Returns:** `Promise<TReturn>` — the query's return value, typed from the Convex function reference.

```ts [server/api/tasks.get.ts]
import { api } from '#trellis/api'

export default defineEventHandler(async (event) => {
  return await serverConvexQuery(event, api.tasks.list, { status: 'active' })
})
```

## `serverConvexMutation(event, fn, args?, options?)`

Run a protected Convex mutation from a Nitro server handler.

**Returns:** `Promise<TReturn>` — the mutation's return value.

```ts [server/api/tasks.post.ts]
import { api } from '#trellis/api'

export default defineEventHandler(async (event) => {
  const body = await readBody<{ text: string }>(event)
  return await serverConvexMutation(event, api.tasks.create, body)
})
```

## `serverConvexAction(event, fn, args?, options?)`

Run a protected Convex action from a Nitro server handler. Use this for handlers that hit external services (Stripe, email providers) or need non-reactive execution.

**Returns:** `Promise<TReturn>` — the action's return value.

```ts [server/api/billing/refund.post.ts]
import { api } from '#trellis/api'

export default defineEventHandler(async (event) => {
  const { chargeId } = await readBody<{ chargeId: string }>(event)
  return await serverConvexAction(event, api.billing.refund, { chargeId })
})
```

## `ServerConvexOptions`

Shared options across the three helpers above.

| Option                        | Type                                                         | Purpose                                                      |
| ----------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `auth?`                       | `'auto' \| 'required' \| 'none' \| 'trusted'`                | Auth mode for the call (default: `'auto'`)                   |
| `authToken?`                  | `string`                                                     | Explicit Convex auth token — skips session cookie resolution |
| `caller?`                     | `{ subject: Subject } & Record<string, unknown>`             | Forwarded caller identity; used with `trusted`               |
| `actingFor?`                  | `{ subject: Subject; reason?: string; grantedBy?: Subject }` | Optional represented identity                                |
| `identityForwardingKey?`      | `string`                                                     | Advanced trusted-call key override; prefer env config        |
| `identityForwardingEnvelope?` | `{ purpose?: IdentityForwardingPurpose; jti?: string }`      | Operation/transport plumbing; app routes usually omit it     |

### Auth mode behavior

| Mode       | Behavior                                                              |
| ---------- | --------------------------------------------------------------------- |
| `auto`     | Use the session cookie when available, call unauthenticated otherwise |
| `required` | Fail the call if auth can't be resolved                               |
| `none`     | Never attach auth — for public Convex handlers only                   |
| `trusted`  | Use the identity-forwarding path for server-to-server calls           |

`auth: 'trusted'` signs a `_trellisForwarding` envelope for the Convex call.
The forwarded `caller` and `actingFor` live inside that envelope, not as
identity-shaped public args. The helper reads the identity-forwarding secret from
the configured environment unless `identityForwardingKey` is explicitly supplied.
Use the env var for normal app routes; per-call keys are for advanced integration
boundaries.

### Failure behavior

- Missing Convex URL configuration throws a `ConvexCallError`.
- Auth resolution failures surface with error context attached.
- Convex error codes, status, category, and issues are preserved when present.

## `createServerConvexCaller(event, options?)`

Request-scoped caller for handlers that make multiple Convex calls — avoids repeating `event` and options on each line.

**Returns:** an object with `query`, `mutation`, and `action` methods.

| Method                | Purpose                                    |
| --------------------- | ------------------------------------------ |
| `query(fn, args?)`    | Run a query with the caller's auth mode    |
| `mutation(fn, args?)` | Run a mutation with the caller's auth mode |
| `action(fn, args?)`   | Run an action with the caller's auth mode  |

```ts [server/api/dashboard.get.ts]
import { createError } from 'h3'
import { subject } from '@lupinum/trellis/auth'
import { api, internal } from '#trellis/api'
import { createServerConvexCaller, delegateToUser } from '#trellis/server'

function getAutomationUserId() {
  const userId = process.env.AUTOMATION_USER_ID
  if (!userId) {
    throw createError({
      statusCode: 500,
      statusMessage: 'AUTOMATION_USER_ID is not configured.',
    })
  }
  return userId
}

export default defineEventHandler(async (event) => {
  const id = getRouterParam(event, 'id')
  if (!id) {
    throw createError({ statusCode: 400, statusMessage: 'Missing dashboard id.' })
  }

  const convex = createServerConvexCaller(event, {
    auth: 'trusted',
    caller: {
      kind: 'service',
      serviceId: 'dashboard-automation',
      subject: subject.service('dashboard-automation'),
    },
    actingFor: await delegateToUser({
      userId: getAutomationUserId(),
      allow: true,
      reason: 'dashboard automation read',
    }),
  })
  const [runbook, incidents] = await Promise.all([
    convex.query(internal.runbooks.getForAutomation, { id }),
    convex.query(api.incidents.recent, { limit: 5 }),
  ])
  return { runbook, incidents }
})
```

::caution
The caller defaults to `auth: 'auto'`, matching the per-call helpers. If a route forwards a `caller`, use the trusted path and pass `auth: 'trusted'` plus that forwarded identity. Forwarded principals are for verified server-to-server paths, not ordinary browser-backed requests.
::

## What this surface is not

::warning
These helpers aren't a second authorization model. They change how a Nuxt server request reaches Convex — they don't replace Trellis guards, appIdentity resolution, tenant boundaries, or backend authorization. Every call still hits the handler's full protection path.
::
