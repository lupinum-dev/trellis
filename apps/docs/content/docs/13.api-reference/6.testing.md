Reference index for the public testing entrypoints. For usage flow, see the [testing section](/docs/testing/getting-started).

## Main exports

| Export                     | Purpose                                                  |
| -------------------------- | -------------------------------------------------------- |
| `convexTestConfig`         | Vitest config wrapper with the Convex edge-runtime setup |
| `createConvexTestModules`  | Module bridge for the generated Convex server            |
| `convexServerMock`         | Generated-server mock bridge used by Trellis apps        |
| `createTestContext`        | Per-test seed + caller API                               |
| `createObservationCapture` | Assert against emitted semantic events                   |

## Scope

This surface exists to make protected-handler and integration-style Trellis tests realistic without rebuilding runtime boundaries by hand. If a test needs to reach around Trellis to work, that's a signal it's at the wrong layer.
