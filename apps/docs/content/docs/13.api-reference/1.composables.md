Reference for the runtime composables you'll look up most often. For task flow use the guides; for the full generated inventory see [API surface](/docs/api-reference/api-surface).

## `useConvexQuery`

Default query composable ‚Äî SSR-aware, hydrates to a live subscription.

```ts
useConvexQuery(query, args?, options?)
```

### Options

| Option                           | Default                                   | Purpose                                         |
| -------------------------------- | ----------------------------------------- | ----------------------------------------------- |
| `server?: boolean`               | Module `query.server` (default `true`)    | Run during SSR                                  |
| `subscribe?: boolean`            | Module `query.subscribe` (default `true`) | Attach live subscription                        |
| `default?: () => T \| undefined` | ‚Äî                                         | Fallback before first real result               |
| `transform?: (input) => output`  | ‚Äî                                         | Map raw result to UI data                       |
| `keepPreviousData?: boolean`     | `false`                                   | Keep last settled result during arg transitions |

### Return

| Field       | Type                                                  |
| ----------- | ----------------------------------------------------- |
| `data`      | `Ref<T \| null>`                                      |
| `error`     | `Ref<Error \| null>`                                  |
| `pending`   | `Ref<boolean>`                                        |
| `status`    | `Ref<'pending' \| 'success' \| 'error' \| 'skipped'>` |
| `isStale`   | `Ref<boolean>`                                        |
| `refresh()` | Force refetch                                         |
| `clear()`   | Reset state                                           |

### Semantics

- Nullish args skip execution; `status === 'skipped'`.
- `keepPreviousData` keeps the previous result visible while `pending` stays `true`.
- `subscribe: false` becomes a one-shot HTTP-backed read.
- Normal query execution failures surface through `error` and `status === 'error'`; setup or configuration misuse may still throw.
- State stays bound to the current args ‚Äî not whichever request settles last.

### Example

```ts
const { data, status, pending, isStale } = await useConvexQuery(
  api.tasks.list,
  { status: 'active' },
  { keepPreviousData: true },
)
```

## `useConvexPaginatedQuery`

Paginated variant for cursor-based list queries.

```ts
useConvexPaginatedQuery(query, args?, options?)
```

### Options

| Option                        | Default        | Purpose                           |
| ----------------------------- | -------------- | --------------------------------- |
| `initialNumItems?: number`    | `10`           | Initial page size                 |
| `server?: boolean`            | Module default | Run during SSR                    |
| `subscribe?: boolean`         | Module default | Attach subscription               |
| `default?: () => Item[]`      | ‚Äî              | Fallback items                    |
| `transform?: (results) => TR` | ‚Äî              | Map results                       |
| `keepPreviousData?: boolean`  | `false`        | Keep last page during transitions |

### Return

| Field                | Type                                |
| -------------------- | ----------------------------------- |
| `results`            | `ComputedRef<Item[]>`               |
| `status`             | `ComputedRef<PaginatedQueryStatus>` |
| `isLoading`          | `ComputedRef<boolean>`              |
| `isStale`            | `ComputedRef<boolean>`              |
| `isExhausted`        | `ComputedRef<boolean>`              |
| `hasNextPage`        | `ComputedRef<boolean>`              |
| `error`              | `Readonly<Ref<Error \| null>>`      |
| `loadMore(numItems)` | Fetch next page                     |
| `refresh()`          | Re-fetch from start                 |
| `reset()`            | Clear pagination state              |

### Constraints

- Use only with Trellis-supported paginated query refs.
- Pass nullish args to skip the whole pagination surface cleanly.

### Example

```ts
const { results, hasNextPage, loadMore, status } = await useConvexPaginatedQuery(
  api.messages.list,
  { channelId },
  { initialNumItems: 20 },
)
```

## `useCachedQuery`

Seeds a detail query from an already-loaded parent query ‚Äî instant render on list-to-detail navigation.

```ts
useCachedQuery(query, args, {
  from: { query, args, find },
  // ...useConvexQuery options
})
```

### Options

| Option                       | Purpose                                                |
| ---------------------------- | ------------------------------------------------------ |
| `from.query`                 | Already-loaded source query                            |
| `from.args`                  | Args used for the source query cache key               |
| `from.find(items)`           | Extract the detail value from the cached source result |
| All `useConvexQuery` options | Applied to the detail subscription                     |

### Return

- Full `useConvexQuery(...)` surface.
- `isFromCache: ComputedRef<boolean>` ‚Äî is the current value seeded from the parent cache.
- `cacheStatus: ComputedRef<'matched' | 'match-missing' | 'source-missing'>` ‚Äî why the seed did or did not apply.

## `useConvexMutation`

Default client-side write path for normal state changes.

```ts
useConvexMutation(mutation, options?)
```

### Options

| Option                                   | Purpose                           |
| ---------------------------------------- | --------------------------------- |
| `optimisticUpdate?: (ctx, args) => void` | Apply cache changes synchronously |
| `onSuccess?: (result, args) => void`     | Runs after server confirms        |
| `onError?: (error, args) => void`        | Runs on failure                   |
| `validate?: ValidateOption`              | Client-side args validation       |

### Return

| Field             | Type                                                       |
| ----------------- | ---------------------------------------------------------- |
| Callable function | `(args) => Promise<result>`                                |
| `data`            | `Ref<Result \| undefined>`                                 |
| `status`          | `ComputedRef<'idle' \| 'pending' \| 'success' \| 'error'>` |
| `pending`         | `ComputedRef<boolean>`                                     |
| `error`           | `Ref<Error \| null>`                                       |
| `reset()`         | Clear tracked state                                        |

### Semantics

- Create the composable synchronously; `await` the returned function call.
- The returned promise rejects on failure after updating `error` and running `onError`.
- Latest request wins for the tracked state.
- Development warns on silent failures if `.error` is never observed.

### Example

```ts
const createTodo = useConvexMutation(api.todos.create, {
  onSuccess: () => toast.add({ title: 'Created', color: 'success' }),
})

await createTodo({ title: 'Ship docs' })
```

## `useConvexAction`

Tracked wrapper for Convex actions. Use it when the backend work needs action semantics ‚Äî third-party calls, longer-running side effects, anything that shouldn't run inside a mutation.

```ts
useConvexAction(action, options?)
```

### Options

| Option                               | Purpose                       |
| ------------------------------------ | ----------------------------- |
| `onSuccess?: (result, args) => void` | Runs after the action returns |
| `onError?: (error, args) => void`    | Runs on failure               |
| `validate?: ValidateOption`          | Client-side args validation   |

### Return

| Field             | Type                                                       |
| ----------------- | ---------------------------------------------------------- |
| Callable function | `(args) => Promise<result>`                                |
| `data`            | `Ref<Result \| undefined>`                                 |
| `status`          | `ComputedRef<'idle' \| 'pending' \| 'success' \| 'error'>` |
| `pending`         | `ComputedRef<boolean>`                                     |
| `error`           | `Ref<Error \| null>`                                       |
| `reset()`         | Clear tracked state                                        |

### Semantics

- Same sync-vs-await rule as mutations: create the composable synchronously, `await` the returned function call (see [`useConvexMutation`](#useconvexmutation) semantics).
- The returned promise rejects on failure after updating `error` and running `onError`.
- No `optimisticUpdate` ‚Äî actions don't have a mutation queue to reconcile against.
- Latest request wins for the tracked state.

### Example

```ts
const chargeCard = useConvexAction(api.billing.charge, {
  onError: (error) => toast.add({ title: error.message, color: 'error' }),
})

await chargeCard({ amountCents: 1500, cardId })
```

## `useConvexAuth`

Primary auth-state composable. Available only when `trellis.auth` is enabled.

```ts
useConvexAuth()
```

### Return

| Field              | Type                                     |
| ------------------ | ---------------------------------------- |
| `sessionUser`      | `Readonly<Ref<AuthSessionUser \| null>>` |
| `isAuthenticated`  | `ComputedRef<boolean>`                   |
| `isPending`        | `Readonly<Ref<boolean>>`                 |
| `isAnonymous`      | `ComputedRef<boolean>`                   |
| `isSessionExpired` | `ComputedRef<boolean>`                   |
| `authError`        | `Readonly<Ref<Error \| null>>`           |
| `refreshAuth()`    | Re-sync Convex auth state                |
| `signOut()`        | Sign out                                 |

### Semantics

- Auth state is pre-populated during SSR when session cookies are available.
- `refreshAuth()` re-synchronizes Convex auth state ‚Äî it doesn't guarantee a logged-in session exists.
- `sessionUser` is a provider session profile and intentionally has no app user id.
- Use `useBetterAuthClient()` for direct Better Auth sign-in and provider actions.

## `useBetterAuthActions`

Tracked wrapper for Better Auth client calls. Available only when `trellis.auth`
is enabled.

```ts
const authAction = useBetterAuthActions()
await authAction.execute(() => client.signIn.email(payload), { redirectTo: '/' })
```

### Return

| Field       | Type                                                       |
| ----------- | ---------------------------------------------------------- |
| `execute()` | `(fn, options?) => Promise<result \| undefined>`           |
| `status`    | `ComputedRef<'idle' \| 'pending' \| 'success' \| 'error'>` |
| `pending`   | `ComputedRef<boolean>`                                     |
| `data`      | `Readonly<Ref<T \| undefined>>`                            |
| `error`     | `Readonly<Ref<Error \| null>>`                             |
| `reset()`   | Clear tracked state                                        |

### Semantics

- On success, `execute()` refreshes Convex auth, stores `data`, and redirects.
- On failure, it stores `error`, sets `status` to `error`, and resolves `undefined`.
- It does not reject like `useConvexMutation()` or `useConvexAction()`.

## `useConvexUpload`

High-level upload composable with queue, progress, cancellation, retry-safe bookkeeping.

```ts
useConvexUpload(generateUploadUrlMutation, options?)
```

### Options

| Option                    | Default        | Purpose                                         |
| ------------------------- | -------------- | ----------------------------------------------- |
| `maxConcurrent?`          | Module default | Concurrent upload slots                         |
| `continueOnError?`        | `true`         | Keep pending items running after one file fails |
| `allowedTypes?: string[]` | ‚Äî              | Client-side MIME filter                         |
| `maxSizeBytes?: number`   | ‚Äî              | Client-side size filter                         |
| `onSuccess?`              | ‚Äî              | Per-file success hook                           |
| `onError?`                | ‚Äî              | Per-file error hook                             |
| `onProgress?`             | ‚Äî              | Per-file progress hook                          |
| `onQueueIdle?`            | ‚Äî              | Queue empty hook                                |

### Return

| Field             | Type                           |
| ----------------- | ------------------------------ |
| Callable          | `upload(fileOrFiles, args?)`   |
| `data`            | `Ref<string \| undefined>`     |
| `status`          | `ComputedRef<UploadStatus>`    |
| `pending`         | `ComputedRef<boolean>`         |
| `progress`        | `ComputedRef<number>`          |
| `error`           | `Readonly<Ref<Error \| null>>` |
| `items`           | `Ref<UploadQueueItem[]>`       |
| `cancelItem(id)`  | Cancel specific file           |
| `cancelAll()`     | Cancel the queue               |
| `clearFinished()` | Clear completed items          |
| `reset()`         | Full reset                     |

### Constraints

- The upload-url mutation must return the signed upload URL as a raw string.
- `allowedTypes` and `maxSizeBytes` run client-side before upload dispatch (advisory ‚Äî still enforce server-side).
- Passing one `File` uses single-upload mode and resolves to one `storageId`.
  Passing a `File[]` or `FileList` uses queue mode and resolves after the batch
  settles. If any file fails, the batch promise rejects even when
  `continueOnError` lets other files continue.

## `useConvexStorageUrl`

Small wrapper over `useConvexQuery(...)` for storage IDs.

```ts
useConvexStorageUrl(getUrlQuery, storageId)
```

### Behavior

- Returns a `ComputedRef<string | null>`.
- Skips automatically when `storageId` is nullish.
- Refreshes reactively when the storage ID changes.
- The query still owns authorization. It should load the app-owned file or
  attachment record and verify caller access before calling
  `ctx.storage.getUrl(...)`.
