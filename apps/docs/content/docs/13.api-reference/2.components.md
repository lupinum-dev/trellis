Auth boundary components, globally registered when `trellis.auth` is enabled. They're page-shell helpers — not a replacement for route protection or backend authorization.

| Component                 | Renders when                        |
| ------------------------- | ----------------------------------- |
| `<ConvexAuthenticated>`   | Auth resolved, user authenticated   |
| `<ConvexUnauthenticated>` | Auth resolved, user unauthenticated |
| `<ConvexAuthLoading>`     | Auth still pending                  |
| `<ConvexAuthError>`       | Auth resolved to an error state     |

## `<ConvexAuthenticated>`

Renders its default slot only when auth resolved and the user is authenticated. Silent during the pending phase.

```vue
<ConvexAuthenticated>
  <Dashboard />
</ConvexAuthenticated>
```

## `<ConvexUnauthenticated>`

Renders its default slot only when auth resolved and the user is unauthenticated. Silent during pending — prevents login UI flashing before auth bootstrap finishes.

## `<ConvexAuthLoading>`

Renders its default slot only while auth state is still pending. Good for loading shells or skeletons.

## `<ConvexAuthError>`

Renders only when auth has failed or ended in error. The default slot receives recovery props.

### Slot props

| Prop              | Type                                           |
| ----------------- | ---------------------------------------------- |
| `retry()`         | Re-run auth refresh without a full page reload |
| `retrying`        | `boolean`                                      |
| `error`           | Raw error                                      |
| `structuredError` | `{ message, ... }`                             |
| `isRecoverable`   | `boolean`                                      |

### Example

```vue
<ConvexAuthError v-slot="{ retry, retrying, structuredError }">
  <p>{{ structuredError?.message }}</p>
  <button :disabled="retrying" @click="retry">
    Try again
  </button>
</ConvexAuthError>
```

::note
`<ConvexAuthError>` catches explicit auth failures and token-decode-style failures. It's for recoverable auth-state UI — not for business authorization errors. Those belong in handler error surfaces.
::

## When to use these components

| Good use                                         | Not a substitute for                   |
| ------------------------------------------------ | -------------------------------------- |
| Page-level auth shells                           | `definePageMeta({ convexAuth: true })` |
| Clear authenticated vs. unauthenticated branches | Backend guards                         |
| Pending auth loading state                       | Resource-level authorization           |
