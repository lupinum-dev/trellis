# 04 Community Courses

Concept brief only. Not runnable yet.

Inspired by: **Skool**

## Why this example exists

This example pressures the intersection of:

- auth
- membership
- billing-aware access
- moderation
- events
- gated learning content

It is a strong test of whether Trellis can power community software, not just internal team tools.

## What Trellis must make easy

- organizations, groups, cohorts, and roles
- billing-aware access to spaces or course modules
- public marketing pages + protected member areas
- community moderation actions with auditability
- event and discussion models on the same tenant runtime

## Agent story

Agents should be able to:

- onboard new members
- answer questions from visible course content
- summarize discussion sentiment
- flag moderation issues
- draft announcements without bypassing human review

## What this example validates

- Better Auth as a real foundation
- paid-access and role layering
- moderated social content
- safe assistant workflows inside a member app
