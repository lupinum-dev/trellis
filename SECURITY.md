# Security Policy

## Supported Versions

Trellis is currently in the `0.1.x` release line. Security fixes target the
latest published `0.1.x` version unless a release note says otherwise.

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability.

Until GitHub private vulnerability reporting is enabled for the repository,
report security issues by contacting the maintainer privately through the
repository owner profile. Include:

- affected package and version,
- a minimal reproduction or affected code path,
- expected impact,
- whether the issue is already public.

The maintainer will acknowledge the report, investigate privately, and publish a
fix with release notes when appropriate.

## Maintainer Audit Notes

The deeper internal security inventory remains in
[meta/SECURITY.md](./meta/SECURITY.md). That file is maintainer context, not the
